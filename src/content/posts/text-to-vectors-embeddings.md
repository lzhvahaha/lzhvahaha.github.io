## 这份笔记讲什么

文字是符号，神经网络只吃数字。把词变成数字（向量化），是进入 Transformer 的第一步。这份笔记从"为什么需要向量化"讲起，遍历人工定义特征 → One-hot 编码 → Word2Vec 训练 → 现代端到端嵌入矩阵的完整演进，帮你理解：最终的嵌入矩阵只是一个随机的二维数组，但它经历了多少弯路才变回这么简单的模样。

## 核心脉络

低维向量信息表达能力不足 → 升维让数据更线性可分 → 人工定义每个维度的特征含义不现实 → One-hot 太稀疏且无法度量相似度 → Word2Vec 用正负样本训练让词向量自动学习语义 → Word2Vec 静态且非端到端的局限 → Transformer 时代：嵌入矩阵随机初始化，随整个模型一起端到端训练。

## 逐节精讲

### 第一节：为什么要向量化

**这一节在解决什么问题**

文字是离散符号，神经网络只能做数值运算。在把文字喂给网络之前，必须先解决一个基础问题：用什么样的数字表示一个词？

**核心思路**

用一个数字表示一个人（比如 1 号、2 号），只能在一维数轴上排列。如果男女混在一起，一维空间里根本找不到一条直线把它们干净分开——得画一条非常扭曲的曲线才行。但如果给每个人两个数字（二维向量），就很容易用一条直线把男女分开。**高维向量让信息表达更灵活，也让分类问题更容易线性可分。**

类比：一维是把所有人排成一队，二维是把他们散开在广场上，三维是整个楼层——维度越高，"站的位置"越多，不同属性的人越容易隔开。

**关键概念展开**

向量从低维升到高维叫**升维**，反过来叫**降维**。一个词如果只用一个数字表示，灵活度远远不够——这个道理和用一维数轴分不开男女一样。所以需要用一个**维度足够多的向量**来表示每个词。

但在进入向量化之前，先确定最小单位。现代大模型处理的最小意义单元叫 **token**，不一定是单个字——可能是一个词、一个子词。把句子切成 token 的过程叫**分词（tokenize）**。切完后，每个 token 在计算机里就是一个数字 id，这个 id 可以在嵌入矩阵中作为索引找到对应的向量。

**这一节留下的悬念**

知道了每个词需要一个高维向量，但这个向量的每个维度分别代表什么？是人来定义还是让机器自己学？

### 第二节：从人工特征到 One-hot 的挫折

**这一节在解决什么问题**

向量的每个维度应该放什么值？最容易想到的办法是人工定义每个维度的"含义"，但事实证明这条路行不通。

**核心思路**

人工定义特征的做法：给向量的每个位置赋予一个人类理解的含义。比如第 1 个位置表示"食物属性"——苹果很高，喜欢几乎没有，我可能稍有一点。第 2 个位置表示"动物属性"，等等。

在二维空间类比，这样定义的向量能让语义相近的词（苹果和香蕉，都是水果）距离很近；同时苹果也和小米（都是电子设备品牌）靠近。但"苹果"到底是水果还是手机，取决于上下文——静态的人工定义根本无法覆盖一个词在不同语境下的多义性。

**关键概念展开**

人工定义特征的问题是死路：每个词的含义和变化太丰富了。就算真有神人硬着头皮填完了所有维度，每个维度的含义也会极其抽象、人类无法理解。这条路不现实。

另一个极端是 **One-hot 编码（独热编码）**：向量的维度等于整个词表大小（比如 10 万），每个词只有一个位置是 1，其余全是 0。"独热"的意思就是只有一个位置是热的。这个方法有两个致命问题：

1. **维度太高**：词表多大向量就多大，计算量爆炸
2. **无法度量相似度**：任意两个不同词的 One-hot 向量都正交，点积永远为 0。"苹果"和"香蕉"的相似度，跟"苹果"和"银河系"的相似度是一样的——都是 0，这显然不合理。

**这一节留下的悬念**

人工定义不行，One-hot 也不行——那能不能让向量自己去"学"出每个维度的值？

### 第三节：Word2Vec——让词向量自己学

**这一节在解决什么问题**

不走极端，找一个中间路线：向量的每个维度先随机初始化，然后通过训练任务让这些值自动调整，最终使语义相近的词在向量空间中距离也近。

**核心思路**

Word2Vec 的核心思想可以用一句话概括：**一个词的含义，由它周围的词决定。** 如果"苹果"和"香蕉"经常出现在相似的上下文中（比如前面是"吃"，后面是"很甜"），那么它们的词向量就会被训练得彼此靠近。

类比：你不认识某人，但发现他身边的朋友你全都认识——那你也大致知道他是哪类人了。词也是通过"身边的朋友"来定义自己的。

**关键概念展开**

Word2Vec 的训练过程：

1. 把所有词的词向量拼接成一个大矩阵——纵向是词表大小，横向是每个词的向量维度。一开始这些值全是随机的。

2. 准备好大量文本句子作为语料，先分词，然后选择一个**中间词**，其余作为**上下文词**。

3. 训练目标：让中间词的词向量与上下文词的词向量的**点积越大越好**（正样本，表示相关）。同时随机选一些不相关的词作为**负样本**，让点积越小越好。

4. 在大量语料的正负样本训练后，这个大矩阵就被训练好了——语义相近的词，向量在高维空间中自然靠近。

训练好的词向量会表现出令人惊艳的效果：把词向量降维到三维空间观察，能发现相似的词聚在一起。还会出现 "man − woman ≈ king − queen" 这样的神奇语义运算——向量的加减法居然对应着语义的变换，说明性别这个"方向"被编码进了向量空间的几何关系中。

> 课程中简化了 Word2Vec 的细节：实际上中心词和上下文词分别用了两个矩阵，但理解核心思想时不必纠结于此。

**Word2Vec 的两个问题**

第一，每个词的词向量是**静态的、固定的**，完全不考虑上下文。"吃了一个苹果"和"买了一个苹果手机"——同一个"苹果"，词向量一样，但含义完全不同。Word2Vec 无法区分。

第二，训练出的词向量矩阵往往作为下游任务的**固定输入**，不是端到端训练的。也就是说，下游模型在训练时不能同时调整词向量，有一种"割裂感"。

**这一节留下的悬念**

Word2Vec 证明了"让向量自己学"是可行的，但它静态且非端到端——能不能把词向量和下游模型揉在一起，一并训练？

### 第四节：端到端嵌入矩阵——最朴素的样子

**这一节在解决什么问题**

随着算力提升和 Transformer 架构的出现，人们发现：与其费尽心机预训练一个词向量矩阵再喂给下游模型，不如直接把它作为整个模型的第一层，随机初始化，跟着整个模型一起端到端训练。

**核心思路**

嵌入矩阵就是一张大表——纵向是词表大小（比如 50000），横向是每个词向量的维度（比如 512）。一开始全是随机数。输入一个 token id（比如 135），就去矩阵的第 135 行取出一整个向量——这就是这个词的**词嵌入（token embedding）**。然后把向量送入 Transformer 后续层。

随着整个 Transformer 做预测、计算损失、反向传播，嵌入矩阵的值也一起被更新。训练结束后，这个词向量才真正学会了表达语义。

类比：嵌入矩阵就像一个还没填名字的班级座位表。每个学生（token）来报道时得到一个随机座位（随机向量）。经过一学期的相处（训练），坐在讲台边的可能是班长（某些维度值高），坐在角落的可能是内向的同学（另一些维度值低）。而且换一个班（换一个训练任务），同样的学生可能会坐到完全不同的位置。

**关键概念展开**

从代码实现的角度看，整个嵌入矩阵的创建只需要一行 PyTorch 代码：

```python
embedding = nn.Embedding(vocab_size, embedding_dim)
```

- `vocab_size`：词表大小，比如 50000
- `embedding_dim`：每个词向量的维度，比如 512

前向传播时，传入 token id 列表（比如 `[1, 3, 5]`），`nn.Embedding` 会自动从矩阵中取出对应行的向量返回——就是这么简单。

**演进脉络总结**

```
人工定义每个维度特征含义 → 不可行，词的含义太丰富
         ↓
One-hot 编码（词表大小向量，一个位置为 1）→ 维度太高，无法度量相似度
         ↓
Word2Vec（正负样本训练，中心词预测上下文）→ 可以学语义，但静态且非端到端
         ↓
Transformer 时代：嵌入矩阵随整个模型端到端训练 → 最简单也最有效
```

这种演进反映了一个更深的规律：随着数据和算力的增长，原来人们费尽心力设计的精巧结构，在"暴力出奇迹"的时代反而成了模型的阻碍。不如什么都不管，一股脑端到端训练来得更好。谷歌 Gemini 3 的多模态完全端到端训练也是同样的思路——不再分文字塔、视觉塔，一切从零一起学。

**这一节留下的悬念**

词嵌入搞定了，但现在的词向量仍然是"和上下文无关"的——"苹果"无论出现在什么句子里，取出来都是同一个向量。怎么让词向量根据上下文动态变化？答案藏在 Transformer 的自注意力机制中，那是下一步要探索的内容。

## 关键术语表

| 术语 | 直觉理解 | 所在章节 |
|------|---------|---------|
| 向量化 | 把词变成一串数字，让神经网络能处理 | 第一节 |
| Token | 模型处理的最小语义单元，可能是字、词或子词 | 第一节 |
| 分词 (tokenize) | 把句子切成 token 的过程 | 第一节 |
| 升维 / 降维 | 增加或减少向量维度，高维更容易线性可分 | 第一节 |
| One-hot 编码 | 只有"自己"那个位置是 1，其余全 0，稀疏且无法衡量相似度 | 第二节 |
| 词向量 | 表示一个词的实数向量，维度不高不低 | 第三节 |
| Word2Vec | 用正负样本训练，让词向量通过"周围词"来学习语义 | 第三节 |
| 词空间 | 所有词向量所在的高维空间，语义近的词距离也近 | 第三节 |
| 嵌入矩阵 | 所有词向量排成的大矩阵，行=词表大小，列=向量维度 | 第四节 |
| Token id | 每个 token 在词表中的编号，作为嵌入矩阵的索引 | 第四节 |
| nn.Embedding | PyTorch 中创建嵌入矩阵的一行代码 | 第四节 |
| 端到端训练 | 嵌入矩阵不从别处拿，跟着整个模型一起训 | 第四节 |
