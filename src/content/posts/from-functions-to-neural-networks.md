## 这份笔记讲什么

本课程回答一个问题：**从最朴素的想法"用函数描述世界"，一路走到现代大语言模型的底层架构 Transformer，中间到底发生了什么？** 五节课，每节 8-10 分钟，把二十多年的 AI 演进压缩为一条清晰的推理链。学完你将理解神经网络是什么、怎么训练、为什么有效、以及 CNN / RNN 分别解决了什么问题——为后续理解 Transformer 和大语言模型打下基础。

## 核心脉络

符号主义试图用精确函数描述世界 → 发现写不出来 → 连接主义转向"猜一个近似函数" → 用线性变换叠加激活函数构造复杂曲线（神经网络） → 损失函数衡量猜得好不好 → 梯度下降 + 反向传播让参数自动调优 → 训练中遇到过拟合/梯度消失等难题 → 工程手段逐个击破 → 矩阵化抽象让公式简洁、利用 GPU 加速 → 全连接层处理图像太浪费 → 卷积层提取局部特征（CNN） → 全连接层处理序列无法体现顺序 → 隐藏状态逐词传递信息（RNN） → RNN 串行慢且丢失长期记忆 → 引出 Transformer 一眼看全局。

## 逐节精讲

### 第一节：从函数到神经网络

**这一节在解决什么问题**

这节是整个系列的基石：在动手做任何计算之前，先回答"我们到底在做什么"——AI 的本质是把世界建模为一个函数，但这条路走不通之后，人类换了什么新思路？以及这个新思路用什么结构实现？

**核心思路**

想象你拿到一堆 x 和 y 的数据点，想知道它们之间的关系。符号主义的做法是盯着数据推导公式，像解数学题一样"看出来" y = 2x。但现实世界的问题——比如"识别这张图是不是猫"——人类根本写不出精确的公式。**连接主义的回答是：写不出来就猜。** 先随便画一条线，然后看着数据一点一点挪动它，直到"差不多对"为止。

**关键概念展开**

早期人工智能走的是**符号主义**路线，认为世间万物都能找到背后明确的规律，用精确的函数表达。这条路在面对图像识别、机器翻译这类"写不出公式"的问题时走到头了。

**连接主义**换了个思路：不再追求精确解，而是寻找一个足够接近真实答案的近似解。方法就是先猜一条直线 y = wx + b，然后不断微调 w 和 b，让这条线越来越贴近数据。求不求精确无所谓，大差不差就行。这个思路虽然听起来"不正经"，但用很少的参数就实现了手写数字识别，证明极其有效。

然而直线能表达的规律有限。数据点散落成曲线状时，怎么调 w 和 b 都够不着——这时需要让线"弯"起来。方法很简单：在线性函数的最外层，再套一个非线性运算。比如外面套个平方、sin 或指数函数。这个套在外层的非线性运算就叫**激活函数**。它的作用是把原本只能画直线的函数"盘活"，变成能画曲线的函数。

一层激活不够灵活？那就再套一层。把"线性变换 + 激活函数"当作一个整体，在这个整体之上再做一次线性变换 + 激活函数，无限套娃下去。理论上，这样可以**逼近任意连续函数**——不管真实规律多复杂，理论上都能拟合。

这种层层嵌套如果写成公式会非常冗长，所以人们发明了**神经网络的画法**：每个圆圈叫一个**神经元**，左边一列是**输入层**（输入变量 x），中间是**隐藏层**（被包在函数内部、看不见），右边是**输出层**（最终结果 y）。信号从左到右依次计算过去，这个过程叫**前向传播**——本质上就是一步步算出函数的值。

> 讲师特别提醒：不要把神经元和生物神经元类比，它们只是恰好用了同一个名字，强行类比反而影响理解。

**公式**

最简单的形式：

$$\hat{y} = g(wx + b)$$

- **x**：输入（比如一张图片的某个像素值），一个数字
- **w**：权重，控制这个输入对输出的影响有多大，是我们要"猜"的参数
- **b**：偏置，相当于整条线的上下平移量，也是要猜的参数
- **wx + b**：线性变换，在坐标系里就是一条直线
- **g(·)**：激活函数，套在外面把直线掰弯
- **ŷ**：预测输出

这个公式的计算就是：拿输入 x，乘上权重 w，加上偏置 b，再经过激活函数 g 处理一下，得到预测值 ŷ。

**这一节留下的悬念**

神经网络的结构搭好了，但怎么确定那几百上千个 w 和 b 到底是几？靠肉眼是调不过来的——我们需要一种自动计算参数的方法。这正是下一节要解决的问题。

### 第二节：计算神经网络的参数

**这一节在解决什么问题**

上一节搭好了神经网络这个"猜函数"的框架，留下了核心问题：几十万甚至上亿个 w 和 b，怎么自动算出来？这节引入三个关键机制：用**损失函数**衡量猜得怎么样，用**梯度下降**指示往哪个方向猜，用**反向传播**高效计算出每个参数该怎么调。

**核心思路**

想象你在漆黑的山里，想走到海拔最低的谷底。你看不到整个山的地形，但你能感觉到脚下的坡度——脚往哪个方向踩，地面是往下沉的。**梯度下降**就是这个过程：每一步都沿着最陡的下坡方向走一小步，走多了自然就到了谷底。损失函数就是这座山的高度，参数 w、b 就是你的位置坐标。

**关键概念展开**

首先要回答一个问题：什么叫"猜得好"？直觉上，预测值和真实数据越接近越好。把这个直觉翻译成数学，就是把每个数据点的预测值 ŷ 与真实值 y 之间的差距算出来，全部加起来作为总误差。这个衡量误差的函数就叫**损失函数** L。

差距用绝对值不好算——绝对值函数不平滑，求导时要分类讨论。换成**平方**，一举两得：消除了不平滑，还放大了大误差的影响（差 5 的贡献是 25，差 1 的贡献只有 1）。最后除以样本数 n 消除样本量差异。这就得到了最常用的损失函数**均方误差 MSE**。

对于最简单的线性回归（y = wx 且只有几个数据点），可以把数据代入 MSE 公式、化简成关于 w 的二次函数，然后令导数等于零直接解出 w。这就是**线性回归**——机器学习中最基本的分析方法。

但神经网络太复杂了，它的损失函数往往是一个有无数山峰山谷的复杂曲面，无法像一元二次函数那样"一步解出"。人们的对策简单粗暴：**一点点试**。当前参数是 (w=5, b=5)，损失是 10。把 w 调到 6，损失变成 9——方向对了。把 b 调到 6，损失变成 11——方向错了，往回调。

w 变化一点点，损失函数变化多少？这就是损失函数对 w 的**偏导数**。所有参数的偏导数组成一个向量，叫**梯度**。沿着梯度的**反方向**走，就是损失函数下降最快的方向。每步走多远由**学习率** α 控制——太大容易跳过最优解，太小收敛太慢。这个沿着梯度反方向不断更新参数的过程，叫**梯度下降**。

还剩最后一个障碍：神经网络的函数嵌套了好几层，直接求损失函数对最底层参数的偏导数非常困难。但巧妙的是，虽然整体复杂，层与层之间的关系却很简单。可以**逐层求导然后乘起来**。就像计算齿轮传动：想知道第一个齿轮转一圈会让最后一个齿轮转几圈，就把每个齿轮的传动比乘起来。这种逐层求导的方式叫**链式法则**。

实际训练中的流程：从右往左（输出层 → 输入层）逐层求偏导，每算出一层的梯度就更新那一层的参数。梯度值像波浪一样从后往前传过来，所以叫**反向传播**。

**公式拆解**

梯度下降的更新规则：

$$w := w - \alpha \cdot \frac{\partial L}{\partial w}$$

- **w**：某个参数（权重），当前值
- **α**：学习率，控制更新步长，典型值 0.001 ~ 0.1
- **∂L/∂w**：损失函数对 w 的偏导数，表示"w 变一点，损失变多少"
- **整体含义**：把 w 往让损失变小的方向挪一小步，挪多少由梯度和学习率共同决定

链式法则（以简化的三层网络求 ∂L/∂w₁ 为例）：

$$\frac{\partial L}{\partial w_1} = \frac{\partial L}{\partial \hat{y}} \cdot \frac{\partial \hat{y}}{\partial a} \cdot \frac{\partial a}{\partial w_1}$$

每个因子都是单独一层里简单的求导，乘起来就得到了最终偏导数。

**一次完整训练** = 前向传播（输入 → 层层计算 → 输出预测） + 反向传播（计算梯度 → 逐层更新参数）。重复多轮，直到损失函数足够小。

**这一节留下的悬念**

训练机制建好了，但在真实数据上跑起来，马上会遇到各种麻烦：训练集上完美，新数据上一塌糊涂；深层网络梯度传着传着就没了……下一节专门对付这些"训练事故"。

### 第三节：调教神经网络的方法

**这一节在解决什么问题**

上一节教会了神经网络怎么训练，但训练不等于好用。你会发现一个诡异的现象：模型在训练数据上表现得完美无缺，一碰到没见过的新数据就稀烂。更大的网络反而更差？这节系统梳理训练中的各种"翻车"场景及其解法。

**核心思路**

训练神经网络就像教一个学生。如果他只是把习题集的答案背下来了，遇到新题就不会做——这叫**过拟合**。我们需要的不是"背题高手"，而是真正理解规律的"好学生"——这叫**泛化能力**。防过拟合的方法，本质上都是在逼模型去学真正的规律，而不是死记训练数据的每个细节。

**关键概念展开**

**过拟合**的根本原因是模型太复杂而数据太简单：模型把训练数据里的噪声和随机波动也当规律学了。识别方法是看训练集和验证集的表现差距——训练集上几乎零误差，验证集上误差很大，就是过拟合。

最简单的预防手段是**增加数据量**。数据足够多，噪声就被淹没了。如果收集不到更多数据，可以在现有数据上"做手脚"——对图像旋转、翻转、裁剪、加噪，创造出新样本。这叫**数据增强**，一举两得：既增加了数据量，又训练模型对各种小变化不敏感（增强了**鲁棒性**）。

**简化模型复杂度**也很直接：别一上来就上最大的模型，简单的线性模型有时反而比复杂网络效果好。

从训练过程本身入手，最天真的方法叫**提前终止**：看验证集误差不再下降就停，不追求训练集上的完美拟合。虽然粗糙但有效。

更精细的控制方法是在损失函数里加**惩罚项**：参数变大的同时，损失函数也跟着变大，这就抑制了参数"野蛮生长"的冲动。用参数绝对值之和做惩罚的叫 **L1 正则化**，参数越大惩罚越狠；用参数平方和做惩罚的叫 **L2 正则化**，对特别大的参数抑制更强。惩罚项前面乘一个**正则化系数** λ 来控制力度——这类控制"控制参数的参数"统称**超参数**。

还有一种简单到离谱但极其有效的方法叫 **Dropout**（由深度学习之父 Hinton 提出）。想象神经网络是一支军队，每次战斗（训练轮次）随机让一部分士兵（神经元）"休假"。这样模型就不能依赖某几个"超级士兵"，必须让每个士兵都有战斗力。结果就是参数之间的依赖被打破，泛化能力大大提升。

除了过拟合，训练中还有其他常见问题：

- **梯度消失**：网络越深，梯度反向传播时一层层乘下去，越来越小，前面几层几乎得不到更新。就像传话游戏，传了五十个人后最初的消息已经面目全非。**残差网络（ResNet）** 通过"跳跃连接"让梯度绕过中间层直接传回去，缓解了这个问题。
- **梯度爆炸**：梯度过大，参数更新幅度失控——合理初始化和**梯度裁剪**可以解决。
- **收敛慢 / 震荡**：梯度下降像小球滚下山，遇到坑坑洼洼可能来回弹跳。**动量法**给更新加上惯性，**Adam 优化器**则自适应地为每个参数调整学习率，让训练更快更稳。
- **计算开销大**：全部数据算一次梯度太慢。**mini-batch** 把数据分成小批，每批算一次更新，大大降低单次开销。

**这一节留下的悬念**

各种工程技巧让训练变得可控，但目前的神经网络结构（全连接层）在处理图像和序列时，还有结构性的浪费——下一节开始动网络结构本身。

### 第四节：从矩阵到 CNN

**这一节在解决什么问题**

前几节默认的神经网络中，每一层所有神经元都和上一层所有神经元相连——这叫**全连接层**。但当输入是一张 30×30 的图片时，900 个像素全连接到下一层 1000 个神经元，参数直接冲到 90 万。更致命的是，把所有像素平铺展开，邻居像素之间的空间关系全丢了——图片往右平移一格，所有神经元的输入都变了，但图片本身几乎没变。这节引入**卷积神经网络（CNN）**，专治这两个毛病。

**核心思路**

CNN 的核心思想用一句话概括：**用一个"小窗口"在整张图片上滑动，每滑到一个位置就提取一组特征。** 这个小窗口就是**卷积核**。因为同一个窗口滑遍整张图——无论在左上角还是右下角，都用同一套参数——所以参数量大幅减少。又因为窗口看到的始终是"一小块邻居像素"，天然保留了空间局部关系。

类比：想象你拿一个手电筒照一张大地图，手电筒只能照亮巴掌大一块。你一格一格地挪动手电筒，每次记录下这块区域"有没有公路、河流、建筑"。走完整张图，你就有了整张地图的特征摘要——这就是卷积在做什么。

**关键概念展开**

在讲 CNN 之前，先把所有公式升一级：把繁琐的逐个加减乘除写法，统一成矩阵形式。一个全连接层的计算可以写成：

$$A^{[l]} = g(W^{[l]} \cdot A^{[l-1]} + b^{[l]})$$

这一方面让公式简洁抽象便于深入讨论，更重要的是——矩阵运算可以用 GPU 并行计算，训练和推理快几个数量级。

**全连接层**的问题：除了参数量爆炸，还有个结构性问题。对于图像识别，人眼识别猫靠的是"圆耳朵、胡须、鼻子"这些局部特征，跟它们在画面的左上角还是右下角无关。全连接层把每个像素当作独立的输入，完全失去了"这些像素在空间上是邻居"的信息。

**卷积运算**的做法：取一个固定大小的矩阵（比如 3×3）叫**卷积核**，把它盖在图像上的某个区域，对应位置的数值两两相乘再求和，得到一个值。然后卷积核滑动一格，再算一次。滑完整张图，得到一张新的"特征图"。

在传统图像处理中，卷积核是人工设计的——特定数值的卷积核能实现模糊、锐化、提取边缘等效果，就是 PS 里的滤镜。但在深度学习中，卷积核的值是**训练出来的**，和其他参数一样通过梯度下降自动调优。

把全连接层替换成卷积层后，神经网络结构变成：**输入 → 卷积层 → 池化层 → … → 卷积层 → 池化层 → 全连接层 → 全连接层 → 输出**。池化层的作用是对特征图降维——比如取每个 2×2 区域的最大值（最大池化）——减少计算量同时保留主要特征。这种结构就叫**卷积神经网络（CNN）**。

CNN 还有一个迷人之处：可以可视化中间层的卷积核到底学到了什么。你会发现浅层卷积核学会检测边缘和纹理，深层卷积核学会检测眼睛、轮子等语义部件——虽然都是隐藏层的事，但居然能看出实际意义。

**公式对比**

全连接层用的是标准**矩阵乘法**（dot product），卷积层用的是**卷积运算**（cross-correlation）。在公式上，就是换了一种运算方式而已，但效果天差地别。

**这一节留下的悬念**

CNN 解决了图像问题，但它有一个根本局限——只适合**静态数据**。如果输入是一句话、一段语音、一个视频（有时间先后顺序的序列数据），CNN 就无能为力了。我们需要一种能"记住前面发生了什么"的网络结构。

### 第五节：从词嵌入到 RNN

**这一节在解决什么问题**

从图像转到自然语言处理的第一个障碍：文字是符号不是数字，神经网络只吃数字。这节先解决"如何把词变成数字"（编码），再解决"如何让网络理解词的顺序"（RNN），最终引出 Transformer 的动机。

**核心思路**

把文字变成计算机能处理的数字，这个过程叫**编码**。两种极端都不好用——用一个数字代表一个词（"我"=1,"地球"=3568）维度太低、无法衡量语义相似度；用词表大小的向量、每个词只有一个位置是 1（one-hot），维度太高且词之间正交无关。折中方案是**词嵌入**：用一个不高不低的向量表示每个词，每个维度的值不是人定的，而是**训练出来的**——语义相近的词，向量也自然靠近。

有了向量表示之后，RNN 的核心创新是引入一个**隐藏状态 h**，在逐个处理词语时把前文的信息"捎"给后面的词。就像读一句话时你脑子里始终记得上文在说什么——h 就是这个短期记忆。

**关键概念展开**

**编码方式对比**：单一数字标识（如 "我"=1, "你"=2）维度太低，且数字大小本身对语义没有意义。"我"是 1、"你"是 2，不表示"你"是"我"的两倍。**One-hot 编码**：词表有 10 万个词，每个词就是一个 10 万维的向量，只有一个位置是 1 其余全是 0。维度太高且每个向量之间互相正交（点积为 0），词和词之间仍然没有关联。

**词嵌入**取中庸之道：设一个适中的维度（比如 300 维），每个词对应一个 300 维的实数向量。关键是这个向量是训练出来的——不是人手工赋值的。语义相近的词，它们的向量在高维空间中距离就近。可以用**点积**或**余弦相似度**来量化这种相关性。所有词向量排成一个大矩阵叫**嵌入矩阵**，每个词向量所在的 300 维空间叫**词空间**。一个有趣的训练效果：训练好的词嵌入中可能出现 "king − man + woman ≈ queen" 这样的向量运算，说明语义关系被编码进了向量空间的几何关系中。

> 经典训练方式有 Word2Vec 等（课程未展开），可以暂时理解为"已经有了一个训练好的嵌入矩阵"。

有了词向量之后，把一句话的每个词都变成向量，平铺开一股脑送进全连接层？有两个问题：输入层随句子长度变化（变长输入）；词的先后顺序信息完全丢失——"狗咬人"和"人咬狗"变成了完全相同的输入。

RNN 的解法：一次只处理一个词，但让前一个词的"记忆"传递给后一个词。具体做法是每个时间步 t 计算两次：

$$\begin{aligned} h_t &= g(W_{xh} \cdot x_t + W_{hh} \cdot h_{t-1} + b_h) \\ y_t &= W_{hy} \cdot h_t + b_y \end{aligned}$$

- **xₜ**：第 t 个词的词向量（比如 300 维）
- **hₜ₋₁**：上一个时间步的隐藏状态，携带着前 t−1 个词的"记忆"
- **Wxh**：处理当前词向量的权重矩阵
- **Whh**：处理前一步隐藏状态的权重矩阵
- **hₜ**：当前时间步的隐藏状态，准备传给下一个词
- **Why**：从隐藏状态到输出的权重矩阵
- **yₜ**：当前词的输出（比如褒贬判断）
- **g**：激活函数

和经典神经网络相比，公式只多了一项 Wₕₕ · hₜ₋₁——就是把"前面说了什么"作为一个额外的输入加进来。但正是这个小小改动，让网络拥有了**处理序列的能力**。

**RNN 的问题**也很致命。第一个问题：**长期依赖丢失**。信息在时间步之间一层层传递，越远衰减越厉害。第 1 个词的隐藏状态传到第 50 个词时，最初的信息几乎消失了。"我在法国长大，所以我会说一口流利的____"——网络需要记住"法国"才能填"法语"，但中间隔了太多词，RNN 可能已经忘了。LSTM 和 GRU 通过门控机制改善了这个问题，但只是缓解，没有根治。

第二个问题：**无法并行**。每个时间步的计算依赖上一步的隐藏状态，必须 h₁ 算完才能算 h₂。这意味着不能利用 GPU 的并行优势，处理长序列极慢。

**这一节留下的悬念**

有没有一种方案，既能捕捉序列中任意两个词之间的关系（无论隔多远），又能完全并行计算？这就是 **Transformer** 的动机——它用**自注意力机制**替代了 RNN 的串行依赖，"一眼"看到整个序列中所有词对之间的关系。课程系列至此为 Transformer 扫清了全部前置知识——虽然视频只到这里，但路线已经完整了。

## 关键术语表

| 术语 | 直觉理解 | 所在章节 |
|------|---------|---------|
| 符号主义 | 用精确公式表达世界规律，像解数学题 | 第一节 |
| 连接主义 | 写不出精确公式就"猜"一个近似函数 | 第一节 |
| 激活函数 | 套在线性函数外面，把直线掰弯 | 第一节 |
| 隐藏层 | 被包在神经网络内部、看不到输入输出的中间层 | 第一节 |
| 前向传播 | 信号从左到右通过各层算出最终结果 | 第一节 |
| 损失函数 | 衡量预测值和真实值的差距，越小越好 | 第二节 |
| MSE（均方误差） | 最常用的损失函数，算平方差再平均 | 第二节 |
| 梯度 | 损失函数在每个参数方向的"坡度"，告诉你往哪走会下降 | 第二节 |
| 梯度下降 | 沿着梯度反方向一步步走向损失函数的最低点 | 第二节 |
| 学习率 | 每步走多大，太大跳过最优点，太小收敛太慢 | 第二节 |
| 链式法则 | 把复杂函数的求导拆成每层单独求导再乘起来 | 第二节 |
| 反向传播 | 从右到左逐层计算梯度并更新参数 | 第二节 |
| 过拟合 | 训练数据完美，新数据一塌糊涂——像只背答案的学生 | 第三节 |
| 泛化能力 | 对未见过数据的表现力，真正想要的东西 | 第三节 |
| 数据增强 | 对现有数据旋转/翻转/加噪来"制造"新数据 | 第三节 |
| L1 / L2 正则化 | 在损失函数里加惩罚项抑制参数变大（L1 用绝对值，L2 用平方） | 第三节 |
| Dropout | 每次训练随机"关掉"一部分神经元，让模型不依赖少数参数 | 第三节 |
| 超参数 | 控制"控制参数的参数"，如学习率、正则化系数 | 第三节 |
| 梯度消失 / 爆炸 | 深层网络中梯度太小（无法更新）或太大（失控） | 第三节 |
| 全连接层 | 每个神经元连接上一层所有神经元，参数量爆炸 | 第四节 |
| 卷积核 | 在图像上滑动的小矩阵，提取局部特征 | 第四节 |
| CNN | 用卷积层+池化层替代全连接层，专为图像设计的网络 | 第四节 |
| 词嵌入 | 用中等维度的实数向量表示词，向量通过训练得到 | 第五节 |
| One-hot 编码 | 词表大小的向量，只有一个位置为 1 其余为 0 | 第五节 |
| 嵌入矩阵 | 所有词向量排成的大矩阵，每列一个词的向量 | 第五节 |
| RNN | 通过隐藏状态在时间步之间传递信息，处理序列数据 | 第五节 |
| 隐藏状态 h | 网络在每个时间步保留的"短期记忆"，承载前文信息 | 第五节 |
| Transformer | 用自注意力替代 RNN，并行处理全序列，大语言模型的基石 | 第五节 |

  ↓                  ↓
