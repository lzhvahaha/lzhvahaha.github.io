## 这份笔记讲什么

Transformer 是现代大语言模型的底层架构。这份材料用一个翻译任务（把"用毒毒毒蛇"翻成英文）贯穿始终，从文字如何变成向量、模型如何让每个词相互"看见"、到逐词生成译文，完整走一遍 Transformer 的全流程。学完你能讲清：自注意力机制在算什么、为什么要多头、残差和归一化起什么作用、编码器和解码器如何分工、最后怎样把向量变回一个真实的词。

## 核心脉络

句子中每个字的含义依赖整体语境 → 需要让每个词相互关联 → 输入预处理（BPE 分词 → 512 维词向量 → 加位置编码）→ 自注意力（Q/K/V 三矩阵，Q 与所有 K 点积算相似度，除 √dk，softmax 成权重，对 V 加权求和）→ 单一关注太局限 → 多头（拆 8 个 64 维并行算再拼接）→ 单层处理不了长文 → 多层堆叠 → 梯度消失 → 残差 F(x)+x 保通路 + 归一化稳数值 → 前馈网络 ReLU 引入非线性 → 编码器（理解输入）→ 解码器（瞻前顾后逐词生成，掩码遮未来，交叉注意力取编码器语义）→ 线性变换映射到词表 → softmax → argmax 输出词 → 循环到结束符 → 问答训练造就大语言模型，GPT 只用解码器。

## 逐节精讲

### 第一节：为什么需要 Transformer——词义依赖整体语境

**这一节在解决什么问题**

机器翻译要解决的核心难题是：同一个字在不同位置含义不同，单拎出来无法判断，必须从整体上看。Transformer 的设计思想正是从这里出发——让句子里的每个词都能相互关联、互为整体。

**核心思路**

材料用"用毒毒毒蛇"这句拗口的话举例：句子里有多个"毒"字，第一个是名词，第二个是动词，含义随位置而变。计算机想准确翻译，必须理解每个字在不同位置的含义，而这必须从整句整体判断——每个字单拎出来都不行。

换句话说，句子中的每个字都和其他字相互关联。计算机想理解这句话，就必须找出这种关联。这正是 Transformer 架构的思想所在。

**这一节留下的悬念**

要找出词与词之间的关联，第一步得先把文字变成模型能计算的数字。怎么变？

---

### 第二节：输入预处理——从文字到带位置的向量

**这一节在解决什么问题**

模型不认字，只认数字。需要把一句话变成一组模型能处理的向量，并且要保留词的先后顺序。

**核心思路**

Transformer 分三步预处理输入：

1. **分词**：用 BPE 等分词工具，把句子拆成最小的语义单位 token。token 不一定是完整的字或词，而是模型词表里登记的基本单元。
2. **词向量编码**：每个 token 被编码成一个 **512 维的词向量**——也就是用一个 512 个数组成的向量来表示一个 token 的语义。
3. **位置编码**：因为所有词向量是**同时**输入模型的，模型本身判断不出它们的先后顺序，所以要给每个词向量额外加一个位置编码，告诉模型"你是第几个"。

**公式直觉**

$$\text{输入向量} = \text{词向量} + \text{位置编码}$$

两个向量都是 512 维，逐元素相加，结果仍是 512 维。加完后，每个向量同时携带"我是谁"和"我在第几个位置"两层信息。

以 10 个 token 的句子为例，最终得到一个形状为 **10 × 512** 的向量组，材料中记作 X。

**这一节留下的悬念**

现在每个词向量都有了身份和位置，但它们彼此还是孤立的——"用"字还不知道句子里有个"毒"字。模型怎么通过 X 找出每个词之间的联系？

---

### 第三节：自注意力机制——Q、K、V 与图书馆类比

**这一节在解决什么问题**

让句子中的每个词都和其他所有词（包括自己）"算一算"关联，从而把全句上下文融进每个词的新表示里。这是 Transformer 最核心的计算。

**核心思路：图书馆类比**

想象你是一个 token，带着你的疑问（Q）走进图书馆，和每本书的索引标签（K）进行匹配。匹配度最高的书，内容最可能解决你的问题，你对它关注度最高，花更多时间读它的内容（V）；关注度低的书则少读。当你读完走出图书馆时，相当于对所有书的知识做了整合——你带走的新知识里，就包含了你对这些书的理解。

通过自注意力计算后的每个 token，都产生了对其他 token 的抽象理解，这种理解就藏在经过加权的输出向量里。

**关键概念展开**

为了让每个词都能"查资料"，模型先用三个权重矩阵 **WQ、WK、WV** 分别和每个词向量相乘，于是每个 token 得到三个向量：

- **Q（查询向量）**：代表当前词"想关注什么"——它要问的问题。
- **K（键向量）**：代表该词"能为其他词提供什么信息""关于什么的信息"——可看作一个标签或索引，供别人来匹配。
- **V（值向量）**：代表该词"实际包含的信息内容"——是真正被检索和聚合的信息本身。

接下来是计算流程：

1. **算相似度**：用每个词的 Q 向量，分别和句子中所有词（包括自己）的 K 向量做**点积**，得到一组原始分数。
2. **缩放**：把结果除以一个常数 **√dk**（dk 是 K 向量的维度），防止点积数值过大导致**梯度爆炸**。这就得到了一组注意力分数。
3. **归一化**：套一个 **softmax**，把分数全部压到 0~1 之间且加起来等于 1，得到一组**注意力权重**——它代表该词和其他每个词的关联程度。
4. **加权求和**：把每个词的 V 向量和它对应的注意力权重相乘，再把每一行加起来（加权求和），就得到一组新的 V 向量。

这组新的 V 向量包含了句子中每个词对其他词含义的理解。材料把整个过程称为**自注意力机制**——顾名思义，自己和自己产生注意力。计算后得到的这组向量记作 **Z**。

**公式直觉**

$$\text{Attention}(Q, K, V) = \text{softmax}\!\left(\frac{QK^T}{\sqrt{d_k}}\right)V$$

- **Q、K、V**：分别是所有词的查询、键、值向量拼成的矩阵，每一行对应一个词。
- **QK^T**：Q 和 K 的转置相乘，本质就是每个 Q 和每个 K 做点积，得到词与词两两之间的相似度矩阵。
- **√dk**：缩放因子，dk 是 K 的维度。点积的数值会随维度增大而变大，除以 √dk 把它压回合理范围，避免 softmax 落到梯度极小的区域。
- **softmax**：把每一行分数变成一组权重（和为 1）。
- **乘 V**：用这组权重对 V 加权求和，得到融合了全句上下文的新向量。

**这一节留下的悬念**

自注意力让每个词都产生了对其他词的理解，但这种理解是"单一"的。现实中一对词之间往往有多方面的语义关系，一次自注意力算得够吗？

### 第四节：多头自注意力——从多个角度理解语义

**这一节在解决什么问题**

单次自注意力只能让一个词对其他词产生**单一**的关注和理解，但语言是多层级的——同一个句子里可能同时存在动词与宾语的关系、代词与指代名词的关系、语义角色、情感、修饰关系等。只算一次太局限。

**核心思路**

Transformer 干脆把最初的输入向量 X 直接拆成多个"头"，让自注意力计算分别独立进行，最后再把结果拼接回来。这样模型能从多个角度整合复杂的语义信息。

**关键概念展开**

材料以 8 头为例：

1. **拆分**：把 512 维的 X 向量拆成 **8 个 64 维**的向量。
2. **独立计算**：8 个头各自独立执行一遍完整的自注意力运算（QK 点积 → 缩放 → softmax → 加权 V 求和）。
3. **拼接**：把 8 个头的计算结果拼起来，重新合并为一个 **10 × 512** 维的向量。

合并后的向量 Z 包含了 8 个头独立计算的结果，能从多个角度整合复杂的语义信息。这整个过程叫**多头自注意力机制**，是 Transformer 中最核心的模块。

**为什么是"拆"而不是"复制"**：512 维总量不变，拆成 8 个 64 维子空间后，每个头在一个更小的子空间里学一种关系模式——就像派 8 个只盯一个维度的专员，各看一面，再汇总。这样总计算量没增加多少，却换来了多视角的表达能力。

**这一节留下的悬念**

经过多头自注意力，一个句子复杂的语义信息已经用向量表示出来了。但单层处理一句话勉强够用，处理一段话、一篇文章就捉襟见肘了。怎么扩展到更深？

---

### 第五节：深层网络的三件套——残差、归一化、前馈

**这一节在解决什么问题**

单层多头自注意力处理长文本能力不足，需要堆多层。但多层神经网络会带来**梯度消失**——训练时梯度无法有效传播，导致只有少数几层真正起作用。本节给出 Transformer 保住深层训练稳定性的三个组件。

**核心思路：残差连接**

梯度消失的一种典型情形：如果某一层训练出的最优解是"什么都不做"，那它的输出 F(x) 就等于 0，而 F(x)=0 会导致梯度信息无法继续传播。

解决办法简单到出奇：把这一层的输出从 F(x) 改成 **F(x) + x**。这样无论 F(x) 是否为 0，总有个 x 存在，相当于提供了一条**直接的、不受权重矩阵影响的梯度通路**。这就是**残差网络**（ResNet 的核心思想）。

**公式直觉**

$$\text{输出} = F(x) + x$$

- **x**：这一层的输入（上一层的输出），绕过 F 直接加到输出上，叫"跳跃连接"或"捷径"。
- **F(x)**：这一层本身学到的变换。
- **F(x)+x**：哪怕 F 学成了 0，梯度仍能沿 x 这条通路回传，深层网络才训得动。

**归一化**

残差之后还要对每层输出的值做**归一化**，让向量中的数值保持在合理范围（材料表述为 0~1 之间），防止出现波动较大的数值。归一化后的输出向量记作 A。它的作用是稳住每层数值的分布，让训练更稳定。

**前馈神经网络（FFN）**

接下来每层加一个前馈神经网络 FFN，对 A 向量的每个元素用 **ReLU 激活函数**做非线性变换。为什么必须非线性？因为之前的多头注意力再怎么复杂，其输出本质仍是**线性**的（矩阵乘法、加权和都是线性操作），线性组合再多次也只是一条直线。ReLU 让这条线"可以拐弯"，从而能拟合任意形状的曲线、逼近任意复杂的函数。

**ReLU 的直觉**：ReLU 的规则是"小于 0 的部分砍成 0，大于 0 的部分原样保留"。就这一个折点，就把一条直线掰成了一个能拐弯的折线；多层堆叠后，折线能逼近任何曲线。

出于相同的目的（稳定训练），前馈网络后面**再加一个残差连接和归一化**。

**这一节留下的悬念**

把"多头自注意力 + 残差 + 归一化 + 前馈 + 残差 + 归一化"叠 N 层，就构成了编码器。它负责理解输入。理解完之后，怎么生成译文？

---

### 第六节：编码器——理解输入

**这一节在解决什么问题**

把前面所有组件按顺序拼起来，定义"编码器"这个模块——它的职责是把输入语言处理成一组富含上下文的向量表示。

**核心思路**

编码器的数据流就是第五节那套结构的 N 层堆叠：

> 多头自注意力 → 残差 + 归一化 → 前馈网络 → 残差 + 归一化 → （重复 N 层）→ 输出向量 x_out

经过这套操作，一个句子中复杂的语义信息就被向量表示了出来。x_out 是编码器的最终输出，它的功能主要是**负责理解输入的语言**。

**这一节留下的悬念**

编码器已经理解了输入句子，下面就要根据这份理解来生成翻译。负责生成的模块叫解码器。

---

### 第七节：解码器——瞻前顾后，逐词生成

**这一节在解决什么问题**

解码器要根据编码器理解到的信息，结合已经生成的词，逐词生成译文。它的核心特征是"瞻前顾后"——"瞻前"看编码器的理解，"顾后"看自己已经生成的序列。

**核心思路**

模型是**逐个**生成单词的：每生成一个新单词，都要把之前已经生成的序列重新输入解码器，以此来预测下一个 token。如果模型还没生成任何 token，就先把一个**起始符**输入进去。

因此解码器收到两个输入：
- 编码器的输出 **x_out**（用于"瞻前"）；
- 模型已经生成的单词序列（用于"顾后"）。

**关键概念展开**

解码器内部三层，依次处理：

**1. 掩码多头自注意力（顾后）**

计算方式和前面的多头自注意力基本一致——对输入的每个 token 算它们之间的注意力权重。比如刚开始只有一个起始符，就让它自己和自己算。

为什么多"掩码"两个字？因为**训练时**为了高效，是把整个答案一次性输入进去的，但必须对解码器中"还未生成的单词"做掩码处理，这样模型就看不到未来的单词，只能根据已生成的单词做预测。每预测出一个结果，就放出一个掩码位置，拿预测和真实结果对比算误差，再据此调整参数——这样训练十分方便。

掩码的具体实现：用 Q_i 和 K_j 计算时，把 **j > i 的未来位置**的注意力分数设为一个**非常大的负数**。softmax 之后这些位置的权重就接近 0，相当于被"遮住"。这么做主要是**保证训练和使用的一致性**，推理时不用再单独调整模型。

（推理时输入解码器的本来就只有已生成的序列，掩不掩码无所谓；但代码中一般仍对未生成位置掩码，保持一致。）

掩码多头自注意力后同样接一个残差连接和归一化，输出记作 **a_mask**。

**2. 交叉注意力（瞻前）**

这一层让 a_mask 和编码器的输出 x_out 之间做注意力计算。和自注意力的唯一区别：向量不是和自己算，而是**和编码器的输出向量算**。

对 a_mask 中的每个 token，让它和 x_out 中所有 token 计算注意力权重并加权——同样是 Q、K、V 那一套，只不过这里的 Q 来自解码器（a_mask），K 和 V 来自编码器（x_out）。

交叉注意力的目的是**融合编码器理解到的的语义信息，实现输入和输出的对齐**——即让生成的每个词都参考原文的完整语义。它和多头自注意力一样也分多头计算，后面同样接残差、前馈、再残差。

**3. N 层堆叠与输出**

经过 N 层这样的结构，最后得到解码器的输出向量 **y_out**。

**这一节留下的悬念**

y_out 还是一个 512 维的向量，怎么把它变成一个真实的英文单词？

---

### 第八节：输出层——从向量到单词

**这一节在解决什么问题**

把解码器输出的 512 维向量 y_out 映射回词表，挑出一个真实的单词输出。

**核心思路**

已知 y_out 维度是 512，而真实单词来自一个固定大小的**词表**（vocab size）。所以需要把 512 维转成词表长度 V。这用一次线性变换实现。

**公式拆解**

$$\text{logits} = y_{out} \cdot W + b$$

- **y_out**：解码器输出向量，维度 512。
- **W**：权重矩阵，形状为 **512 × V**（V 是词表大小）。它把 512 维映射到 V 维。
- **b**：偏置向量，形状为 V。
- **logits**：结果向量，长度为 V，每个值表示对应单词的得分。

接着对 logits 做 **softmax**，把得分转成**概率分布**——prob 向量的每个值表示对应单词的概率是多少。

最后用 **argmax** 找出概率最大的那个单词输出即可。

**生成循环**：以上过程循环执行多次，每次预测下一个 token，并把新词追加回输入，直到输出的单词为**结束符**为止。这就是逐词生成的完整闭环。

**这一节留下的悬念**

到这里，从中文到向量再到英文的全流程就闭环了。Transformer 就这些？它和今天的大语言模型是什么关系？

---

### 第九节：从翻译到大语言模型

**这一节在解决什么问题**

翻译只是 Transformer 的冰山一角。把训练方式换一下，同一个架构就能变成今天的大语言模型。

**核心思路**

如果把训练任务从"翻译"换成"问答"——用问题/答案的方式训练模型，模型就会变成一个问答机器，能回答各种问题。现在主流的大语言模型就是用这种方式训练，从而涌现出各种智能。

根据使用场景不同，架构也会改进：

- **只用解码器**：现在很多大模型（如 GPT）其实**只有解码器、没有编码器**，只专注生成任务——看前面的词，猜下一个词。
- **编码器 + 解码器**：更专注翻译这类需要**对齐**的任务。

Transformer 的出现让自然语言处理乃至整个 AI 迈上了新阶段，搭起了一座从语言到智能的桥梁。

| 自注意力 | 句内每个词和所有词（含自己）算关联，融合全句上下文 | 第三节 |
| 多头注意力 | 拆多个子空间并行算注意力，从多角度理解语义 | 第四节 |
| 残差连接 F(x)+x | 输入直接加到输出上，保住梯度通路，防梯度消失 | 第五节 |
| 归一化 | 把每层数值拉回合理范围，稳住训练 | 第五节 |
| 前馈网络（FFN） | 用 ReLU 做非线性变换，让模型能拟合任意曲线 | 第五节 |
| ReLU | "负数砍 0、正数保留"，给直线加折点引入非线性 | 第五节 |
| 编码器 | 多头注意力+残差+归一化+前馈叠 N 层，负责理解输入 | 第六节 |
| 解码器 | 瞻前顾后逐词生成，含掩码自注意力与交叉注意力 | 第七节 |
| 掩码（Mask） | 把未来位置分数设成大负数，遮住未生成的词 | 第七节 |
| 交叉注意力 | 解码器的 Q 去查编码器的 K/V，让生成参考原文语义 | 第七节 |
| logits | 向量映射到词表后的得分向量，每个值对应一个词 | 第八节 |
| Softmax | 把得分转成概率分布（和为 1） | 第八节 |
| argmax | 取概率最大的那个词输出 | 第八节 |
| GPT | 只保留解码器的 Transformer，专注"猜下一个词" | 第九节 |
